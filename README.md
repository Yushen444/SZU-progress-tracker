# SZUer 培养方案数据库

本项目将深圳大学“全校方案查询”的“查看详情”数据转换为 SZU 通关程序使用的兼容 JSON。网页保留本地 DOC/DOCX/RTF/PDF 解析，同时可直接从云端数据库筛选并导入培养方案。

## 当前数据

- 2025 级：328 个培养方案
- 2024 级：354 个培养方案
- 2023 级：357 个培养方案
- 合计：1,039 个培养方案，110,187 条课程记录

云端索引按年级、学院、修读类型和培养方案分类，并保留每个年级与学院的交叉计数。每条数据库记录使用官网 `PYFADM` 作为唯一 `sourceId`，不按培养方案名称去重，因为同名方案可能属于不同学院。

## 数据边界

- 数据来自官网已发布培养方案的详情接口。
- 一条 JSON 对应一个完整培养方案和一个官网 `PYFADM`。
- 选择主修方案会替换主数据库，并决定首页毕业学分与模块要求。
- 选择辅修或辅修学士学位方案只追加未重复课程，不改变主修毕业要求；首页下方会为每个辅修方案单独显示总学分、在修学分和分项要求进度。
- 主辅修共有课程只需录入一次成绩，但会分别计入各自培养方案的独立进度。
- “管理辅修”可单独移除误加方案，并清理对应课程库条目和已选课程记录。
- 开课单位字段仍保留在课程数组结构中，但课程库界面不展示该列。

## 目录

- `data/raw/plans-list-{grade}.json`：各年级官网清单
- `data/raw/plans/{grade}/{PYFADM}.json`：详情接口原始数据
- `public/plan-db/`：生成后的数据库
- `site/plan-db/`：网页部署副本，由构建脚本自动同步
- `data/validation/plan-db-report.json`：完整性校验报告

## 生成与校验

```powershell
npm run build:plans
npm run validate:plans
npm run test:addon-progress
npm run test:design
```

校验会检查各年级官方数量、清单与原始文件一一对应、方案名称和年级一致、`sourceId` 唯一、生成文件完整、课程分类和毕业要求引用有效。课程数不超过 3 的方案只列为人工复核警告，因为部分辅修学位方案确实可能只有毕业论文课程。

## 本地运行

```powershell
npm run serve
```

默认地址为 <http://127.0.0.1:4173>。

当前本地页面使用“明快效率型”设计：首次打开优先引导导入培养方案，导入后收起引导并进入课程管理与毕业进度页面。本地改版前快照位于 `backups/web-design-taste-20260714-161559/`，恢复命令见该目录的 `ROLLBACK.md`。

此设计版需先在本地确认，未确认前不要执行部署脚本。

## 部署

`yumao_key.pem` 仅用于部署，不要复制、打印或提交私钥正文。部署脚本会先备份线上 `index.html` 和现有 `plan-db`，再上传 `site/index.html` 与 `site/plan-db/`。

```powershell
.\scripts\deploy.ps1
```
